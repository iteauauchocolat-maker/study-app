# 学習アプリ プロジェクト前提

このファイルは Claude Code がセッションをまたいで前提を引き継ぐためのメモ。新しいセッションでは最初にここを読むこと。

## 目的・対象

- 中学生・高校生（オーナーのお子さん2人）向けの学習アプリ
- 英語・漢字・社会・理科などの一問一答が出発点
- 利用者は iPhone Safari（ホーム画面追加で PWA 起動）+ iCloud Drive 運用を想定

## 開発・配信形態

- 開発は **PC とスマホ両方の Claude Code** からバイブコーディング（当初は PC ローカルのみの想定だったが GitHub 経由に変更）
- ソースは GitHub リポジトリ `iteauauchocolat-maker/study-app` で管理
- 配信物は **1 ファイル HTML**（`study-app.html`）+ 外部 `data.json`（将来分離予定）
- API キーをアプリ本体に持たせない（OCR は別経路）

## 現在の実装状況（`study-app.html`）

「第1弾」に相当する全部入りが既に入っている（約 2,180 行 / 74KB、単一 HTML）。

- iOS 風の UI（ライト/ダーク両対応、safe-area 対応、PWA メタタグ）
- プロフィール選択（"中学" / "高校" の 2 名、問題プールは共有・履歴は個別）
- 教科 → 単元 → 一問一答、自己採点（○△×）、結果表示
- 印刷モード（A4/B5、`@media print`、問題用紙／解答付き、問題数指定）
- 問題・教科・単元の管理 UI、JSON のエクスポート/インポート
- データ永続化は `localStorage`（キーは `STORAGE_KEY`）、初回は `buildSampleData()` が 20 問のサンプルを投入
- データモデルは `subjects / units / questions / profiles / history`、`questions.type` と `payload` 相当の構造で将来の選択式・画像対応を想定

## 進め方（合意済みフェーズ）

- 第1弾: 単一 HTML + サンプル 20 問で全部入り（**完了済み**）
- 第2弾: 実機調整、使い勝手の修正
- 第3弾: `data.json` 外部化、エクスポート/インポートの運用フロー確立
- 第4弾: OCR 取り込み運用の確立（当面は Claude Code に画像を渡して `data.json` を更新する案を採用。Claude.ai 経由 / 専用 import HTML は将来案）

## 設計上の注意

- データモデルは将来の拡張（選択式 4 択 / 発音記号・音声 / 図やグラフ画像）に耐える形を維持する
- Safari ITP 対策：PWA 化前提 + 起動時に `data.json` を読み、終了時にエクスポートを促す導線を残す
- iPhone から PDF 書き出し（ネットプリント想定）も視野に入れる
- 解答ゆれ（"徳川家康/とくがわいえやす" 等）に備え `answers` は配列
- 元プリント画像は OCR 失敗時の参照やライセンス確認のため必ず保管する方針

## 開発ルール

- ブランチ: 作業ブランチに開発・コミット・push する。`main` には push しない
- コミット: 明確な日本語メッセージで、論理単位ごとに区切る
- PR: ユーザーが明示的に依頼したときのみ作成する
- セッション開始時はこのファイル + 直近のコミットログを必ず確認する
